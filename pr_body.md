🎯 **What:** The code health issue addressed
Removed a stray `console.log(json)` used for standard output in the `outputJson` function of the drilldown CLI, and replaced it with `process.stdout.write(json + "\n")`.

💡 **Why:** How this improves maintainability
Using `console.log` can sometimes introduce unintended formatting or debugging artifacts. By strictly using `process.stdout.write` with an explicit newline, we ensure that the CLI behaves correctly when piping its standard output to other tools or files without interference.

✅ **Verification:** How you confirmed the change is safe
- Visually reviewed the `outputJson` logic to ensure `json` is a string (which it is, generated by `JSON.stringify`).
- Compiled `@paper-tools/core` and `@paper-tools/drilldown`.
- Ran the `drilldown` test suite via `pnpm --filter @paper-tools/drilldown test`, with all tests successfully passing.

✨ **Result:** The improvement achieved
Cleaned up the standard output generation for the drilldown CLI, increasing reliability when interoperating with other command-line tools.
